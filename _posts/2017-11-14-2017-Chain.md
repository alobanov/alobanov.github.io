> You need to sequentially validate data and then apply transformations to it. The order of validations and transformations should be flexible, depending on the situation. The behavioral pattern [**"Chain of Responsibility"**][5] comes to the rescue.

<br />

For example, we need to validate a phone number string and format it. Let's describe in order the actions we want to perform:

- The string must not be empty or <span class="wordcode">nil</span>
- The string must contain a specific number of characters
- The string must be formatted according to a specified format

> **Chain of Responsibility** is a behavioral design pattern that lets you pass requests along a chain of handlers. Each handler decides whether to process the request itself or pass it further along the chain.

## Implementation

<ol class="custom-counter">
  <li>
  Common interface for all handlers <span class="wordcode">MiddlewareProtocol</span>:
{% highlight swift %}
protocol MiddlewareProtocol {
  // Link current and next handler instances
  @discardableResult func link(with: MiddlewareProtocol) -> MiddlewareProtocol
  // This method must be overridden — it contains all the logic
  func check(value: MiddlewareItem) -> MiddlewareItem
  // Helper method to simplify calling the next check in the chain
  func checkNext(value: MiddlewareItem) -> MiddlewareItem
}
{% endhighlight %}
  </li>
  <li>
  Add the <span class="wordcode">MiddlewareItem</span> enum — this type holds the value itself <span class="wordcode">.value</span> and an error <span class="wordcode">.error</span> if validation fails:
{% highlight swift %}
public enum MiddlewareItem {
  case value(String?)
  case error(String)
}
{% endhighlight %}
  </li>
  <li>
  Base implementation of <span class="wordcode">MiddlewareProtocol</span>:

{% highlight swift %}
class Middleware: MiddlewareProtocol {
  var next: MiddlewareProtocol?

  @discardableResult func link(with: MiddlewareProtocol) -> MiddlewareProtocol {
    self.next = with
    return with
  }

  func check(value: MiddlewareItem) -> MiddlewareItem {
    return checkNext(value: value)
  }

  func checkNext(value: MiddlewareItem) -> MiddlewareItem {
    guard let next = self.next else { return value }
    return next.check(value: value)
  }
}
{% endhighlight %}
  </li>
</ol>

#### All handlers must inherit from Middleware

Now let's write concrete handlers for validating and transforming data — in our case, a string:

1. Handler for <span class="wordcode">nil</span> and empty string checks: <span class="wordcode">CheckNilMiddleware</span>
2. Handler for string length checks: <span class="wordcode">CheckCountMiddleware</span>
3. Handler that formats the string according to a specified format: <span class="wordcode">PhoneFormatterMiddleware</span>

## Breakdown

1. The handler must inherit from <span class="wordcode">Middleware</span>
2. Override the method <span class="wordcode">check(value: MiddlewareItem) -> MiddlewareItem</span> and describe the validation/transformation logic for <span class="wordcode">.value(let val)</span>.
3. If everything succeeds, call the next check in the chain: <span class="wordcode">return checkNext(value: value)</span>
  * To stop execution, immediately return an error: <span class="wordcode">return .error("Example error text")</span>.
  * To substitute data, pass a new value: <span class="wordcode">return checkNext(value: .value("New value"))</span>

{% highlight swift %}
override func check(value: MiddlewareItem) -> MiddlewareItem {
    switch value {
    case .value(let str):
      if str == nil { // Perform check
        // Validation failed, return error
        return .error("Example error text")
      }
    default:
      break
    }
    // If everything is fine, call the next
    // check in our chain
    return checkNext(value: value)
  }
{% endhighlight %}

## Usage

Validate and format a phone number using our implementation:

{% highlight swift %}
let start = Middleware()
start
  .link(with: CheckNilMiddleware())
  .link(with: CheckCountMiddleware(length: 11))
  .link(with: PhoneFormatterMiddleware(format: "+X (XXX) XXX XX-XX"))

let result = start.check(value: .value("79634480209")))

switch result {
case .error(let error): print(error)
case .value(let result): print(result ?? "")
}
{% endhighlight %}

We can add a touch of functional magic with the <span class="wordcode">|></span> operator:

{% highlight swift %}
precedencegroup ForwardOperator {
  associativity: left
}

infix operator |>: ForwardOperator

func |> (lhs: MiddlewareProtocol, rhs: MiddlewareProtocol) -> MiddlewareProtocol {
  return lhs.link(with: rhs)
}

// Then you can write it like this
let start = Middleware()
start
  |> NilValidation()
  |> CountValidation(length: 11)
  |> PhoneFormatterValidation(format: "+X (XXX) XXX XX-XX")

switch start.check(value: .value("79634481259")) {
case let .error(error): print(error)
case let .value(result): print(result ?? "")
}
{% endhighlight %}

The result printed to the console: <span class="wordcode">+7 (963) 448 02-09</span>

## Conclusion

> This way we can add as many independent handlers as we want and use them depending on the situation.

* Image editor. Add handlers that perform manipulations on an image — a convenient way to apply filters.
* Validation during user login or registration.
